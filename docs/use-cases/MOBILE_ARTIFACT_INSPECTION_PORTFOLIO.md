# Mobile Artifact Inspection Portfolio Use Case

## Purpose

DexPilot is a defensive Android DEX inspection tool.

Its role in the portfolio is not to replace mobile applications and not to run inside them. Its role is to inspect Android artifacts generated by mobile applications in a read-only, deterministic and evidence-oriented way.

This document connects DexPilot with two applied mobile projects:

- Movia: urban mobility mobile application.
- Vaultia: local-first digital vault focused on privacy and security.

## Portfolio thesis

The portfolio is structured around secure mobile engineering:

- Movia demonstrates real-world mobile product development.
- Vaultia demonstrates privacy-first and security-first mobile architecture.
- DexPilot demonstrates low-level Android artifact inspection and defensive analysis.

Together, the three projects form a technical line:

    Product mobile app
        ↓
    Security and privacy architecture
        ↓
    Android artifact inspection

## Role of each project

### Movia

Movia is the applied product case.

It demonstrates:

- Mobile app development.
- Android build and release workflow.
- Location-aware features.
- Urban mobility routing.
- Backend integration.
- Privacy and consent concerns.
- Real release evidence.

Movia generates Android artifacts such as APK files. Those APK files contain DEX files, including `classes.dex`.

### Vaultia

Vaultia is the applied security and privacy case.

It demonstrates:

- Local-first architecture.
- Offline-first security decisions.
- Digital vault threat modeling.
- Sensitive data handling.
- Cryptography planning.
- OPSEC-oriented engineering.

Vaultia will also generate Android artifacts. Those artifacts should be inspected as part of the release process because security-focused apps must be especially careful about their compiled output.

### DexPilot

DexPilot is the inspection tool.

It demonstrates:

- DEX binary parsing.
- Header validation.
- Map list parsing.
- String ID parsing.
- Type ID parsing.
- Bounded reads.
- Deterministic text and JSON reports.
- CI-backed release process.
- Evidence-oriented engineering.

DexPilot can be used to inspect DEX files extracted from Movia and Vaultia builds.

## Intended integration model

DexPilot should remain separate from Movia and Vaultia.

The correct integration is through the build and release process:

    Movia APK / Vaultia APK
        ↓
    Extract classes.dex
        ↓
    Run DexPilot analyze
        ↓
    Generate JSON and text report
        ↓
    Store release evidence
        ↓
    Use the result as a mobile artifact inspection checkpoint

DexPilot should not be embedded inside the final applications.

## Current DexPilot capability

As of v0.3.0, DexPilot can inspect:

- DEX magic and version.
- Declared and actual file size.
- Header fields.
- Endian tag.
- DEX section counters and offsets.
- map_list.
- string_ids.
- bounded string_data_item samples.
- type_ids.
- type descriptors resolved through string_ids.
- validation errors.
- text report output.
- JSON report output.

This is enough for structural DEX inspection, but not yet enough for full Android security auditing.

## Future capability needed for stronger Movia and Vaultia audits

To become a stronger artifact inspection tool for Movia and Vaultia, DexPilot should evolve toward:

- proto_ids parsing.
- field_ids parsing.
- method_ids parsing.
- class_defs parsing.
- APK container support.
- AndroidManifest parsing.
- permissions audit.
- hardcoded URL and string inspection.
- sensitive token pattern detection.
- release evidence generation.
- CI security gate mode.

## Example manual workflow

After generating an APK from Movia or Vaultia:

    unzip -p app-release.apk classes.dex > /tmp/classes.dex
    dexpilot analyze --json /tmp/classes.dex > dex-report.json
    python3 -m json.tool dex-report.json

The generated JSON report can be archived as release evidence.

## What this integration does not claim

This integration does not claim that DexPilot fully secures Movia or Vaultia.

It also does not claim complete malware analysis, reverse engineering coverage, APK hardening, vulnerability detection, or cryptographic verification.

The correct claim is:

    DexPilot provides a read-only structural inspection layer for Android DEX artifacts generated by mobile applications such as Movia and Vaultia.

## Portfolio positioning

The combined portfolio message is:

    Movia shows product engineering.
    Vaultia shows security and privacy architecture.
    DexPilot shows Android artifact inspection.

This creates a stronger technical narrative than isolated projects because it connects mobile product development, security thinking and low-level Android analysis.
