# 02 — Objetivos

## Primary Objective

Build a CLI tool capable of reading a `.dex` file, parsing its header, validating its basic structure, and producing a reliable technical report.

## Secondary Objectives

- Keep the architecture testable and extensible.
- Avoid premature optimization or bytecode rewriting.
- Use Kotlin/JVM idiomatically.
- Maintain safe logging practices.
- Use fixtures and automated tests from the first MVP.
- Keep the project suitable for technical portfolio demonstration.

## Portfolio Objective

DexPilot should demonstrate Android ecosystem knowledge beyond UI development, including bytecode structure, binary parsing, CLI tooling, testing, and software architecture.
